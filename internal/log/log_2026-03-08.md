### Muon optimizer via MultiOptimizer (v0.4.0)
- Root cause of MultiOptimizer KeyError: MLX's `tree_unflatten` converts string-digit dict keys ("0", "1") back to lists. When the optimizer's `_split_dictionary` calls `tree_unflatten`, the resulting list nodes cause `tree_map` to try integer indexing against dict state, producing `KeyError: 0`.
- Fix: use non-digit prefixed keys -- `self.layers = {"l0": ..., "l1": ...}` and `self.value_embeds = {"v0": ..., "v1": ...}`.
- Also moved `window_sizes` from `nn.Module` attribute to `GPTConfig.compute_window_sizes()` method -- plain Python lists stored as module attributes pollute the parameter/gradient tree.
- MultiOptimizer with filter-based routing: `is_muon_param(path, weight)` sends 2D+ layer weights (excluding ve_gate) to Muon, everything else to AdamW.
- Separate LR schedules per optimizer group (MATRIX_LR for Muon, EMBEDDING_LR for AdamW).
- Compiled eval for final evaluation pass.
- Fixed warmup step time estimation: skip first 4 steps instead of 2.
- All 8 tests pass (including 2 new MultiOptimizer tests).
